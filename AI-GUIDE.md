## 一、项目概况

这是一个**纯静态资讯展示网站**,部署在 GitHub Pages 上。

- **技术栈**:HTML + CSS + JavaScript(无后端、无框架、无构建步骤)
- **内容来源**:AI 生成与整理的资讯文章
- **数据存储**:文章数据直接内嵌在 `index.html` 的 `newsData` 数组中
- **部署方式**:推送到 GitHub main 分支即自动部署,1-2 分钟生效

### 核心文件

| 文件 | 作用 | AI 是否需要修改 |
|------|------|----------------|
| `index.html` | 网站主文件(含数据和样式) | ✅ 新增/修改文章时改这里 |
| `content/articles/` | 已发布文章的 Markdown 归档 | ✅ 新文章存这里 |
| `content/drafts/` | 草稿区 | ✅ 草稿存这里 |
| `content/sources/` | 原始素材,按分类存放 | ✅ 素材存这里 |
| `data/news.json` | 数据备份(当前未被网站读取) | ⚠️ 可选,建议同步 |
| `templates/article.md` | 新文章模板 | ❌ 一般不改 |
| `README.md` | 仓库说明 | ❌ 一般不改 |
| `AI-GUIDE.md` | 本文件 | ❌ 一般不改 |

## 三、分类体系

网站使用 **5 个固定分类**,文章只能归入其中之一:

| 分类 | 适用内容 | 示例 |
|------|----------|------|
| `科技` | 编程语言、开发工具、技术趋势、AI | Rust 语言、大模型、WebAssembly |
| `财经` | 商业模式、市场分析、行业洞察、投资 | 订阅经济、开源商业、消费趋势 |
| `时事` | 热点事件、社会动态、国际要闻 | 重大事件、政策变化 |
| `生活` | 生活方式、健康、文化、消费 | 消费习惯、文化现象 |
| `观点` | 深度观点、评论、思考 | 技术反思、行业预测 |

**规则**:
- 分类值必须**完全匹配**上述五个字符串(含大小写)
- AI 相关内容归入「科技」
- 产品设计类思考归入「观点」
- 商业模式分析归入「财经」

## 四、newsData 数据结构 ⭐

网站的数据源是 `index.html` 中的 `newsData` 数组。**这是 AI 最需要关注的结构。**

### 位置

在 `index.html` 中搜索 `const newsData = [` 即可定位(约第 657 行)。

### 字段说明

```javascript
{
  id: 1,                    // 数字,唯一 ID,新文章取现有最大 id + 1
  category: "科技",          // 字符串,必须是 5 个分类之一
  title: "文章标题",          // 字符串,显示在卡片和弹窗
  excerpt: "一句话摘要",      // 字符串,卡片摘要(建议 50-80 字)
  date: "2026-07-21",        // 字符串,格式 YYYY-MM-DD
  readTime: "4 分钟",         // 字符串,预估阅读时间
  source: "来源名称",         // 字符串,资讯来源
  content: [                 // 数组,正文内容,按段落拆分
    "第一段正文。",            // 字符串 = 普通段落
    "第二段正文。",
    { quote: "引用金句" },     // 对象 { quote: "..." } = 引用块
    "第三段正文。"
  ]
}
```

### 完整示例(现有文章)

```javascript
{
  id: 1,
  category: "科技",
  title: "大模型推理成本一年下降 90%,应用层创业迎来黄金窗口",
  excerpt: "随着底层模型价格持续走低,AI 应用的试错成本大幅降低。",
  date: "2026-07-21",
  readTime: "4 分钟",
  source: "AI 行业观察",
  content: [
    "过去一年里,主流大模型的 API 调用价格下降了超过 90%。",
    "这意味着过去只有大公司玩得起的 AI 应用,现在一个人也能做。",
    { quote: "当算力像水电一样便宜,真正稀缺的就不再是技术本身,而是用技术解决问题的想象力。" },
    "当然,窗口期不会永远敞开。"
  ]
}
```

### content 数组的两种元素

1. **字符串** → 渲染为 `<p>` 段落
2. **对象** `{ quote: "..." }` → 渲染为 `<blockquote>` 引用块

**不要用其他格式**,只支持这两种。

## 五、操作场景

### 场景 A:新增一篇文章

**步骤**:

1. **确定新 id**:查看 `newsData` 中最大的 id,新 id = 最大值 + 1

2. **生成文章内容**:按以下要求生成
   - 标题:简洁有信息量,15-30 字
   - 摘录:50-80 字,概括核心观点
   - 正文:3-6 个段落,每段 80-200 字,可穿插 1-2 个引用块
   - 阅读时间:按字数估算(300字/分钟)
   - 日期:用当天日期(YYYY-MM-DD)

3. **写入 index.html**:在 `newsData` 数组**末尾**(最后一个 `}` 后、`]` 前)添加新对象。注意:
   - 前一个对象后要有逗号 `,`
   - 新对象末尾不要加逗号(如果是最后一个)

4. **存档 Markdown**:在 `content/articles/YYYY/MM/` 下创建 `YYYY-MM-DD-标题简写.md`,格式见下方「Markdown 文件格式」

5. **推送**:git add → commit → push

**代码示例 —— 添加新文章到 newsData**:

修改前(末尾文章):
```javascript
      {
        id: 8,
        category: "财经",
        title: "开源商业模式的第三次浪潮",
        // ...
      }
    ];
```

修改后:
```javascript
      {
        id: 8,
        category: "财经",
        title: "开源商业模式的第三次浪潮",
        // ...
      },
      {
        id: 9,
        category: "科技",
        title: "新文章标题",
        excerpt: "新文章摘要",
        date: "2026-07-22",
        readTime: "3 分钟",
        source: "来源",
        content: [
          "第一段。",
          "第二段。"
        ]
      }
    ];
```

### 场景 B:修改已有文章

1. 在 `newsData` 中按 `id` 或 `title` 找到文章
2. 修改对应字段
3. 同步修改 `content/articles/` 下的 Markdown 文件
4. 推送

### 场景 C:删除文章

1. 从 `newsData` 数组中删除整个对象(注意修正逗号)
2. 删除 `content/articles/` 下对应的 Markdown 文件
3. 推送

```css

  --rule: #e7e5e4;        /* 边框色 */
  --accent: #0f766e;      /* 主强调色(深青绿) */
  --accent-soft: #ccfbf1; /* 强调色浅版 */
  --accent2: #f59e0b;     /* 次强调色(琥珀) */
  --accent2-soft: #fef3c7;/* 次强调色浅版 */
```

改配色只需改这些变量值,全站自动生效。

### 场景 E:添加新分类

1. 在 `index.html` 的筛选栏添加按钮(约第 574 行):
   ```html
   <button class="filter-btn" data-cat="新分类">新分类</button>
   ```
2. 在页脚分类列表添加对应链接(约第 617 行)
3. 在 `content/sources/` 创建新分类目录
4. 更新 `data/news.json` 的 `categories` 数组
5. 更新 `AI-GUIDE.md` 第三节的分类表
6. 推送

## 八、内容生成规范

当用户要求生成新文章时,AI 应遵循:

### 标题
- 15-30 字
- 有信息量,不要标题党
- 可用数字、对比、疑问增加吸引力

### 摘录(excerpt)
- 50-80 字
- 概括核心观点,不要重复标题
- 用句号结尾,不要省略号

### 正文
- 3-6 个段落
- 每段 80-200 字
- 逻辑清晰:问题→分析→观点→结论
- 可穿插 1-2 个引用块(`{ quote: "..." }`)突出金句
- 语言风格:专业但不晦涩,有观点但不偏激
- 避免空洞套话,每句都要有信息量

### 阅读时间
- 按 300 字/分钟估算
- 向上取整:"3 分钟"、"5 分钟"

### 日期
- 用当天日期
- 格式:YYYY-MM-DD

### 来源(source)
- 若有明确来源,写来源名称
- 若为 AI 原创观点,写「AI 观点」或主题相关名称

## 九、常见错误避免

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 分类名拼错(如「时世」) | 文章不显示在筛选中 | 严格用:科技/财经/时事/生活/观点 |
| newsData 末尾多逗号 | JS 报错,网站白屏 | 最后一个对象不加逗号 |
| content 元素格式错误 | 渲染异常 | 只用字符串或 `{ quote: "..." }` |
| id 重复 | 文章互相覆盖 | 新 id = 最大 id + 1 |
| 忘记同步 Markdown | 归档不完整 | 每篇都存 Markdown 副本 |
| 中文标点混用 | 排版不统一 | 统一用中文标点(,。:?) |
